Efficient Identification of Core and Dead Features in Variability Models

Download this project as a .zip file Download this project as a tar.gz file

Context

Variability models play a key role in product line engineering as they are used to represent the common and variable features that products may include, and what constraints among the features must be satisfied to guarantee the validity of the products. Valuable analysis operations on variability models can be performed by black box reusing logic engines, such as SAT-solvers and binary decision diagram libraries. Unfortunately, such kind of reuse implies long computation times for operations that need calling the engines many times. To overcome this problem, we work on new algorithms that directly deal with the data structure of a binary decision diagram encoding a variability model. In particular, the algorithm published in this site is specifically designed to detect core and dead features

Description

This repository provides an algorithm to identify core and dead features, which is written in the C++ programming language as an extension of the BuDDy BDD package.

How to Install BuDDy

Follow the instructions in this section to install the BuDDy BDD package on your system, if it is not already installed.

  1. Download buddy-2.4.tar.gz (766,5 kB).
  2. Unzip "gunzip buddy-2.4.tar.gz".
  3. Untar "tar xvf buddy-2.4.tar".
  4. "./configure"
  5. "make"
  6. "make install"

BuDDy Documentation

The buddy-2.4/doc directory contains documentation for the BuDDy package in postscript. An introduction to binary decision diagrams is described in the bddnotes.ps file. In addition, you can find a full description of the buddy package in buddy.ps.

How to Install Our Algorithm

Follow the instructions in this section to install our algorithm implementation.

  1. Download the code hperez30-CoreAndDeadFeatures-51bbbae.tar (2,1 kB).
  2. Unzip "gunzip hperez30-CoreAndDeadFeatures-51bbbae.tar.gz".
  3. Untar "tar xvf hperez30-CoreAndDeadFeatures-51bbbae.tar".

    The following directories are created:
  • coreanddead/core/ -> core and features algorithm implementation
  • coreanddead/core_test/ -> a simple test program (use this test as a template to build your programs)
  • coreanddead/bdds/ -> some examples of BDDs that can be loaded and processed with BuDDy and our algorithm
  • readme.txt

Compiling the core_test template

Follow the instructions in this section to compile our test program.

  1. Open a terminal (Ctrl + Alt + T in Ubuntu).
  2. Unzip "gunzip hperez30-CoreAndDeadFeatures-51bbbae.tar.gz".
  3. Go to "coreanddead/core_test/" directory.
  4. Type "g++ -O3 -std=c++0x test.c ../core/core.c -o test -lbdd -I path_to_buddy/src -I../core/"

Running the core_test template

Follow the instructions in this section to run the test program.

  1. Open a terminal (Ctrl + Alt + T in ubuntu).
  2. Go to "coreanddead/core_test/" directory.
  3. Type "./test path_to_bdd_sample_file option_method"
